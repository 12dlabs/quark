Let's start to build a page to show a simple page here. We use Modulex as asynchronous modules loader for example. And you can download Quark typed definition file for your project reference.

Firstly, creates a page with following HTML code. This page should contain following script files.

  • Modulex seed and adapter
<!DOCTYPE html>
<html>
    <head>
        <meta charset="utf-8" />
        <title>Sample</title>
        <script src="https://g.alicdn.com/hi/modulex/5.0.0/seed-debug.js" type="text/javascript"></script>
        <script src="https://g.alicdn.com/hi/quark/1.8.1000/index.js" type="text/javascript"></script>
        <script type="text/javascript">
            <!--
            AliHub.Runtime.ModulexLoader.addModule("local", "./");
            modulex.use("local/demo", function (d) {
                var form = d.form("page_demo");
            });
            //-->
        </script>
    </head>
    <body>
        <div id="page_demo"></div>
    </body>
</html>

Now, we have prepared all files used to load our library. Following are the scripts which we need implement the business logic. A Type Script file demo.js is needed. Firstly, we want to implement a panel to show a customized information, such as a form. We add code into the coding scope in demo.js file.

// Form generator.
export function form(id) {
    var c = new AliHub.Common.BindingControl(id);
    c.viewModel({ id: "001", nick: "Kingcean" });
    c.setTemplate("ajax", "form.htm");
    return c;
} 

Then, we create form.htm file to fill view template.

<div id="__view_control_c_name" data-bind="if: model()">
    <span>ID </span>
    <input data-bind="value: model().id" />
</div>
<div id="__view_control_c_bio" data-bind="if: model()">
    <span>Nickname </span>
    <input data-bind="value: model().nick" />
</div>

And it works now.
