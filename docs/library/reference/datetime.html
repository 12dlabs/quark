There are lots of utilities for date and time.

AliHub.Common.DateTime

This is a powerful tools to provide lots of functions for date and time.

Functions

See comment documentation.

Samples

var qDate = AliHub.Common.DateTime;
var date = new Date(2015, 9, 10, 10, 10);

// Parses the time tick to Date object.
var str1 = qDate.parse(date.getTime());

// Gets a customized string (e.g. 2015-10-10 10:10:00.000).
var str2 = qDate.toCustomizedString(date, "yyyy-MM-dd HH:mm:ss.mmm");

// Gets horoscope string.
var str3 = qDate.getHoroscope(date).symbo;

// Gets time span between two dates in millisecond and formatted string.
var span1 = qDate.getSpan(date, qDate.parse("now"));
var str4 = qDate.getSpanString(date, qDate.parse("now"));
// To get a formatted string of time span from a given date to now, you can also use following way.
var str5 = qDate.getNowSpanString(date)
// You can also add seconds or days to a given date.
date = qDate.addSeconds(date, 200);
date = qDate.addDays(date, 2);

AliHub.Common.Stopwatch

This is a powerful stopwatch to record time span of an action.

Member properties and methods

See comment documentation.

Samples

var sw = new AliHub.Common.Stopwatch();

// You can set some options such as a URL to record the result with a tracking identifier.
sw.trackingId = "15E5E06C-3127-4099-8285-FDC98772CEFB";
sw.notificationOptions = {
    subject: null,
    key: "/api/sample/asset/time.html?tracking={tracking}&channel={channel}&message={message}",
    channel: "test"
};

// Start the timer.
sw.record();

// You can stop it whenever and it will send the response automatically.
// You can set an optional message to mark why it needs stop.
sw.stop("1st");

// You can also resume it. And stop it again.
sw.record(true);
sw.stop("2nd");

// Or record a new one. And stop.
sw.record();
sw.stop("3rd");
