本地化

你可以为不同语言和地区定义不同的现实内容，以更好的支持本地化。 例如，如果需要设计一个欢迎画面，对于简体中文用户，可能需要显示“你好”，而英文用户则应当显示“Hello”。

Quark 提供一种非常简捷高效的方式来实现这一点。你所需要做的事情就是，为当前站点创建一个模板，然后填充各语言包，最后才需要的地方使用它。

创建模板

模板用于存储本地化信息和其它模板内容。创建过程非常简单，只需如下一行代码。在这里，你需要传入一个唯一标识，做为模板的 ID，以和其它模板区分开来。

var template = AliHub.Res.templates("A_Module_Name", true);

你可以将其设计为单例模式，以便在整个 Web App 中随时可以调用它。

语言包

创建一个对象，以添加属性的方式，在里面填充某一中语言的所有文本。

// 创建一个对象，里面每个属性对应一段本地化文本。
// 属性名会在之后做为索引，而属性值则是本地化文本本身.
var res = {
    hello: "你好",
    bye: "再见",
    name: "名称"
};

// 将这个对象注册成为某一个或多个市场的语言包。
template.strings.reg("zh-Hans", res);
template.strings.reg("zh-CN", res);
template.strings.reg("zh-SG", res);

// 以同样方式注册其它语言包。
res = {
    hello: "Hello",
    bye: "Bye",
    name: "Name"
};
template.strings.reg("en", res);

使用

现在，你可以非常简单的开始使用。如下方代码所示。

var helloString = template.localString("hello");

对于简体中文用户，“你好”会被返回；而英文用户则会返回“Hello”。

不过有的时候，你可能需要批量复制出一批本地化文本，例如，当使用 Angular/AngularJs 的时候，你可能需要在 $scope 中的某个属性设置好所有所需要的显示内容。 下方示例将复制出已注册的本地语言包中的两段文本，以属性的方式，赋值给某个给定对象。

var strings = {};
templates.strings.copyFields(strings, ["hello", "bye"]);
