Localization

You can define different strings for different market. For example, a welcome page show Hello for English users and 你好 for simplified Chinese.

It is very easy to do so. You need create a template container firstly. Then define a string set for specific markets to register. To use them, pass the string keys and Quark will help you to select the string from the target market.

Create a template container

Template container is used to store the information including local strings and others. You can create it as following sample. Need a unique module name to identify.

var template = AliHub.Res.templates("A_Module_Name", true);

You can set the template as singleton for all the web application.

Define local strings and register

Create an object to fill the strings with keys. The key is to map a local string. Following is a sample.

// Create an object to fill the strings.
// A property is for a local string.
// The property key is the mapping key.
// The value is the local string.
var res = {
    hello: "Hello",
    bye: "Bye",
    name: "Name"
};

// Register the string set to the template container as one or more markets.
template.strings.reg("en", res);

// And for other market.
res = {
    hello: "你好",
    bye: "再见",
    name: "名称"
};
template.strings.reg("zh-Hans", res);
template.strings.reg("zh-CN", res);
template.strings.reg("zh-SG", res);

Usages

You can use the local string by following sample.

var helloString = template.localString("hello");

It will return Hello for English users and 你好 for simplified Chinese users.

To batch copy a number of properties (especially for Angular/AngularJs developers), you can code as following sample which is to copy 2 of local strings to an object as its properties.

var strings = {};
templates.strings.copyFields(strings, ["hello", "bye"]);
